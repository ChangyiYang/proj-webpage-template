Task 1:

To rasterize triangles, first, we need to find a way to determine whether a point is inside the triangle. To do this, we use the three-line test , which is basically the dot product between the normal vector of the line and the vector from one vertex to the point.

To improve efficiency, we fist find the minimum and maximum of x and y. Then we only check points that are inside the rectangle [ xmin to xmax, ymin to ymax], which is exactly the bounding box of the triangle.

Supersamping is useful because it is a effectively algorithm for antialiasing. In general, it increase the sample rate to meet with the high frequency signal.

To enable supersampling, the major change is that we check sample points with smaller step, the overall for-loop looks like this.

      int n_sqrt = sqrt(sample_rate);
      float step = 1./(sample_rate*2);
      
      // fill the trangle with inside check
      for (int x = x_min_int; x<= x_max_int; x++){
          for (int y = y_min_int; y<= y_max_int; y++){
              for (int i = 0; i < sample_rate; i++){
                  int y_sample = i/n_sqrt;
                  int x_sample = (i+1)% n_sqrt;
                  
                  if (inside(x + (2*x_sample+1)*step, y + (2*y_sample+1)*step)){
                      sample_buffer[(y * width + x)* sample_rate + i] = color;
                  }
              }
          }

As you can see, a for loop for sample rate is added to enable supersampling. Another difference is instead of calling fill_pixel, we now directly write into the sample_buffer.

Task 5:

When we transform form actual coordinates to texture coordinates, the result is always not have a value in texture since texture map, like any picture, is discrete. Therefore, we need a way to choose which texture value to use, that’s why pixel sampling comes up. The nearest method is that simply choose the nearest point to the target. And the bilinear method is a kind of weighted average of the surrounding points of the target.

Since bilinear method is kind of weighted average, which is similar to the idea of supersampling, it should be more antialiasing than nearest method. Following are some comparison.

The bilinear sampling is much more antialiasing than nearest sampling. The nearest sampling with 16 sample rate is similar to bilinear sampling with 1 sample rate. This is because bilinear sampling is a weighted average of surrounding points, which is like doing supersampling in texture map.

Task 6:

Level sampling is particular for texmap architecture. It used to deal with situation that the texmap level we want is not a integer and decide which level of texmap we should go for sampling. To implement texture mapping, first, we need to calculate the desired map depth, which use the gradients. Then, for nearest level sampling, we just choose the nearest available texmap. For linear level sampling, we do a linear combination of two close texmap level.
